# Release Checksums and Reproducible Build Notes

This document defines the minimum repeatable release flow for MAMA artifacts.

## Goals

- Build artifacts from a tagged commit with a clean working tree.
- Keep deterministic artifact names and structure.
- Publish SHA-256 checksum manifests for independent verification.

## Baseline Requirements

- Use Go `1.24.x`.
- Build from a clean checkout (`git status --short` empty).
- Build from a tagged commit on `main`.

## Automated Release Flow

Primary workflow: `.github/workflows/release-artifacts.yml`

Recommended user-facing assets generated by that workflow:

- `MAMA-Setup-Windows.exe`
- `MAMA-macOS.tar.gz` (universal2)
- `MAMA-Linux.tar.gz`

Optional advanced assets are also generated (architecture-specific portable packages).

Release-note automation:

- `.github/workflows/release-notes.yml`

Signing/notarization automation:

- `.github/workflows/release-signing.yml`

## Local Maintainer Packaging

### Windows installer (recommended)

```powershell
powershell -ExecutionPolicy Bypass -File scripts\windows\package-portable.ps1 -OutputDir dist\mama-windows-amd64-portable -Arch amd64 -BinaryName mama.exe
powershell -ExecutionPolicy Bypass -File scripts\windows\package-installer.ps1 -PortableDir dist\mama-windows-amd64-portable -AppVersion v1.0.0
```

### Linux/macOS portable builds

```bash
bash scripts/release/package-portable.sh linux amd64 dist/MAMA-Linux
bash scripts/release/package-macos-universal.sh dist/MAMA-macOS
```

## Checksums

Generate checksums from repository root:

```bash
bash scripts/release/generate-checksums.sh dist
```

Verify:

```bash
sha256sum -c dist/SHA256SUMS.txt
```

On systems without `sha256sum`:

```bash
shasum -a 256 -c dist/SHA256SUMS.txt
```

## Update Manifests

Generate per-artifact update metadata:

```bash
bash scripts/release/generate-update-manifest.sh dist/MAMA-Setup-Windows.exe v1.0.0 https://github.com/<org>/<repo>/releases/download/v1.0.0/MAMA-Setup-Windows.exe
```

This writes a JSON payload with version, URL, checksum, and size.

## Reproducibility Checklist

- Same source commit/tag.
- Same Go line (`1.24.x`).
- Same packaging scripts.
- Matching checksum manifests across rebuilds.

If checksums differ, compare build environment and embedded metadata before publishing.
